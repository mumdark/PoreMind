# OpenClaw 技能：PoreMind 聊天式分析流程

[English](./SKILL.md) | [中文](./SKILL.zh.md)

## 目标

这是给 OpenClaw 使用的**行为技能说明**（不是 Python SDK）。  
用于让 OpenClaw 学会如何按 PoreMind 流程执行可复现分析，并帮助用户输出结论、表格、图像与模型产物说明。

## 核心原则

1. **可重复性优先**
   - 始终记录数据来源/版本、随机种子、参数值与环境信息。
2. **参数级可解释性**
   - 每一步执行前先解释：函数目的、关键参数、默认值、预期影响。
3. **分步骤引导**
   - 严格按流程推进，不跳过关键校验点。
4. **结构化交付**
   - 每个关键步骤都输出：文字结论 + 表格 + 图像建议 + 产物说明。

## 对话行为要求

当用户提出分析请求时，OpenClaw 应当：

1. 先补齐关键信息：
   - 数据路径与格式（ABF/CSV/其他）
   - 分析目标（分类 / 事件探索 / 质量控制）
   - 事件检测方法与预期方向
2. 明确可复现配置：
   - `seed`
   - 软件环境
   - 输出目录与命名规范
3. 按下述流程依次执行。
4. 每一步都输出：
   - **为什么做这一步**
   - **使用参数**（默认值 + 覆盖值）
   - **结果摘要**
   - **下一步建议**

## 标准流程（PoreMind）

### 第1步：数据加载
- 目标：创建分析对象并校验样本-分组映射。
- 校验点：样本数、文件有效性、channel/sweep 可用性。

### 第2步：预处理/降噪
- 目标：使用指定方法进行降噪。
- 必须说明：方法名、关键滤波参数、对波形的影响。
- 校验点：输出降噪前后质量对比表。

### 第3步：事件检测
- 目标：按指定方法提取候选事件。
- 必须说明：检测方法、方向（`up/down`）、阈值/基线参数。
- 校验点：按样本汇总事件数量并标记异常样本。

### 第4步：特征提取
- 目标：将事件转换为可分析特征表。
- 必须说明：特征列表与可选自定义特征函数。
- 校验点：缺失值比例 + 特征分布提示。

### 第5步：事件过滤/质控
- 目标：过滤噪声或低质量事件。
- 必须说明：过滤方法与硬阈值。
- 校验点：输出保留/剔除汇总表。

### 第6步：建模（可选）
- 目标：训练并比较模型（传统模型/DL）。
- 必须说明：训练/验证策略、评估指标、随机种子。
- 校验点：混淆矩阵 + 指标表 + 最优模型选择理由。

### 第7步：新样本推理（可选）
- 目标：对未知样本进行预测。
- 必须说明：模型版本、特征结构要求、置信度解释。
- 校验点：输出含各类别概率的预测结果表。

## 最终输出模板（每次结论都应包含）

1. **结论摘要**（3–7条）
2. **参数表**（步骤 / 参数 / 值 / 设定原因）
3. **结果表**（关键数值）
4. **图像计划或图像解读**
5. **产物清单**（模型路径、表格路径、图像路径）
6. **复现账本**（seed、版本、命令/流程回放）

## 安全与质量规则

- 信息不足时先提问澄清。
- 禁止编造指标或文件。
- 必须提示风险：数据泄漏、类别不平衡、过拟合、事件过少。
- 任一步失败时，必须给出：
  1) 可能原因，
  2) 最小修复方案，
  3) 建议重跑方式。
